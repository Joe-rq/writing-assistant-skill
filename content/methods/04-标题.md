# 04-标题：取标题

> **核心原则**
>
> 1. **数字冲击优先**：具体数字对比 > 技术术语 > 问题描述
> 2. **认知速度差异**：大脑处理数字<100ms，理解术语500-1000ms
> 3. **倍数门槛≥10倍**：2倍不够震撼，10倍才是数量级差异
> 4. **门槛降低**：让不懂技术的人也能感受到"改进很大"
> 5. **服从小白战略**：小白看得懂、想点击

---

## 核心公式

```
标题吸引力 = 数字对比冲击力 × 戏剧性词汇 × 门槛降低度

关键要素：
- 数字对比冲击力：倍数越大，冲击力越强（建议 >10倍）
- 戏剧性词汇："革命性"、"秘密"、"颠覆"等（激发好奇心）
- 门槛降低度：避免直接使用技术术语（降低理解成本）
```

**为什么是乘法？**

- 只有数字没有戏剧性 → 缺乏情绪触发 → 点击率低
- 只有戏剧性没有数字 → 缺乏可信度 → 像标题党
- 数字+戏剧性但门槛高 → 看不懂 → 不点击

---

## 认知科学原理

### 为什么数字对比有效？

**大脑处理速度差异**：

```
视觉皮层识别数字：<100ms
理解技术术语：500-1000ms
理解问题描述：1000-2000ms

数字对比 = 最快触达，最强冲击
```

**具体数字的三重优势**：

1. **视觉显著性**：数字在文字中突出（天然吸引眼球）
2. **认知具象化**：15万→2千 = 可视化的巨大差异
3. **情绪触发**：倍数差异 → "这么大的改进？" → 好奇心

**核心洞察**：

```
用户的大脑对"75倍差异"的反应是本能的（快速、直观）
对"上下文爆炸"的理解是后天的（需要学习、思考）
标题的目标是**最快触发点击**，而非**展示专业性**
```

## 执行步骤

### 第1步：提炼核心数字对比（3分钟）

找到内容中最震撼的**量化改进**：

**检查清单**：

```
□ 是否有前后对比（从X到Y）
□ 差异倍数是否 ≥10倍（数量级差异）
□ 数字是否具体（避免"大幅提升"等模糊表达）
```

**案例**：

```
技术事实：Anthropic的方案将 Prompt Caching 结合上下文压缩技术
核心数据：从 15万Token → 2千Token（75倍压缩）

数字对比：✅ 15万 vs 2千（差异明显）
倍数：✅ 75倍（远超10倍）
具体性：✅ 具体数字，非模糊表达
```

---

### 第2步：选择戏剧性词汇（2分钟）

**三类高效词汇**：

1. **权威背书型**：
   - "Anthropic 官方"、"OpenAI 内部"、"谷歌新方案"
   - 作用：借用权威降低怀疑

2. **戏剧性形容词**：
   - "革命性"、"颠覆式"、"突破性"
   - 作用：放大改进的重要性

3. **悬念词**：
   - "秘密"、"内幕"、"真相"
   - 作用：激发好奇心

**组合公式**：

```
[权威来源] + [戏剧性词汇] + [数字对比] + [悬念词]

例子：
"Anthropic 革命性方案：AI Agent 从 15万Token 干到 2千Token 的秘密"
  ↑           ↑                    ↑                        ↑
权威来源   戏剧性词汇            数字对比                  悬念词
```

---

### 第3步：降低技术门槛（3分钟）

**核心原则**：让不懂技术的人也能感受到"改进很大"

**技术术语替换表**：

| 技术术语 | 门槛高的表达 | 门槛低的表达 |
|---------|------------|-------------|
| 上下文爆炸 | "MCP上下文爆炸怎么办？" | "从15万Token干到2千Token" |
| Token优化 | "Token使用优化方案" | "成本降低98%" |
| 算法创新 | "算法性能提升" | "速度快10倍" |
| 架构改进 | "系统架构升级" | "内存占用降90%" |

**判断标准**：

```
✅ 你妈妈能不能理解这个标题？
✅ 不懂技术的人能不能感受到"改进"？
❌ 如果需要解释术语才能理解 → 门槛太高
```

---

### 第4步：A/B测试优先级

如果有多个标题候选，优先级判断：

```
优先级从高到低：

1. 数字对比 + 戏剧性 + 低门槛
   例："Anthropic革命性方案：从15万Token干到2千Token的秘密"

2. 数字对比 + 低门槛（缺戏剧性）
   例："从15万Token到2千Token：Anthropic的新方法"

3. 戏剧性 + 低门槛（缺数字）
   例："Anthropic革命性方案：AI成本暴降的秘密"

4. 只有问题描述（缺数字+戏剧性）
   例："MCP上下文爆炸怎么办？Anthropic给出了新答案"
```

## 标题模板

### 模板1：数字对比型（首选）

```
从[大数字]到[小数字]：[方法/来源]

例子：
- "从 15万Token 干到 2千Token"
- "从 3小时 缩短到 10分钟"
- "从 100步 简化到 3步"
```

---

### 模板2：痛点前置型

```
[痛点] + [解决方案]

例子：
- "每次都要找提示词？用 Skills 一键搞定"
- "命令行很麻烦？这个工具让你爱上CLI"
```

---

### 模板3：时间数字型

```
[时间] + [动词] + [结果]

例子：
- "3分钟看懂MCP"
- "5个方法搞定数据分析"
- "10分钟学会Claude Code"
```

---

### 模板4：对比型

```
[旧方法] vs [新方法]
或 [不需要X] + [也能Y]

例子：
- "不需要学Python也能做MCP"
- "新方法比旧方法快10倍"
- "从手动复制到一键调用"
```

---

### 模板5：悬念型

```
[现象] + 原来是因为[原因]

例子：
- "为什么你的AI总不听话？问题在这里"
- "MCP入门：3个案例看懂Server开发"
```

## 数字使用技巧

### 技巧1：构建数字对比矩阵

当内容有多个数字时，选择**冲击力最大**的：

```
Anthropic 方案的多个数字：
- Token数量：15万 → 2千（75倍）✅ 最震撼
- 成本降低：98.7%（也很好，但不如倍数直观）
- 响应速度：提升3倍（倍数太小）

选择：15万 → 2千（差异最明显）
```

### 技巧2：数字单位选择

**原则**：选择让差异最明显的单位

```
例子：150,000 vs 2,000

方案1："从 150,000 到 2,000"
方案2："从 15万 到 2千" ✅
方案3："从 150K 到 2K"

选择方案2：
- 中文语境下，"万"和"千"的对比最直观
- 避免过多零（认知负担）
- 避免字母K（部分用户不熟悉）
```

### 技巧3：避免数字陷阱

```
❌ 不要堆砌多个数字：
"从15万到2千，速度提升3倍，成本降低98%"
→ 问题：信息过载，冲击力分散

✅ 聚焦单一数字对比：
"从15万Token干到2千Token"
→ 优势：单一焦点，冲击力集中

❌ 不要用模糊数字：
"大幅降低"、"显著提升"
→ 问题：无法产生具体想象

✅ 用具体数字：
"降低98%"、"提升10倍"
→ 优势：具体、可感知

❌ 不要用小数点：
"从 150,000Token 降到 2,034Token"
→ 问题：不够简洁

✅ 用简洁数字：
"从 15万Token 降到 2千Token"
→ 优势：简洁、记忆成本低
```

## 注意事项

### ✅ 要做的

1. **数字必须真实**：
   - 不能为了冲击力虚构数字
   - 可以选择最震撼的真实数据点

2. **倍数 ≥10倍**：
   - 2倍改进 → 不够震撼
   - 10倍以上 → 数量级差异

3. **戏剧性要有度**：
   - ✅ "革命性方案"（可接受）
   - ❌ "宇宙级突破"（过度夸张）

### ❌ 不要做的

1. **不要堆砌术语**：
   ```
   ❌ "MCP上下文爆炸怎么办？"
   → 问题：术语门槛高（什么是上下文爆炸？）
   → 结果：只有懂的人点，受众窄

   ✅ "从 15万Token 干到 2千Token"
   → 优势：数字直观（75倍！），不需要理解术语
   → 结果：所有人都能感受到"巨大改进"
   ```

2. **不要用模糊表达**：
   ```
   ❌ "大幅提升性能"
   ✅ "速度提升10倍"
   ```

3. **不要过度夸张**：
   ```
   ❌ "改变世界的发现"
   ✅ "节省90%的时间"
   ```

## 适用场景

**高度适用**：
- ✅ 性能优化（速度提升、成本降低）
- ✅ 效率工具（时间节省、流程简化）
- ✅ 技术突破（算法创新、架构改进）

**中度适用**：
- ⚠️ 方法论分享（如果有量化数据）
- ⚠️ 工具测评（如果有对比数据）

**不适用**：
- ❌ 纯观点文章（无数据支撑）
- ❌ 故事叙述（缺乏量化对比）
- ❌ 情感类内容（数字削弱感染力）
