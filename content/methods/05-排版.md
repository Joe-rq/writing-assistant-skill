# 05-排版：公众号适配

> **核心原则**
>
> 1. **三级标题系统**：只用H1/H2/H3，H1可以有多个（标记大章节）
> 2. **行高1.8**：书籍标准，小白长时间阅读不疲劳
> 3. **公众号适配**：有序列表无空格（`1.项目`）
> 4. **换行标记**：用 `---` 产生大间距（公众号 md 转 html 特殊标识）
> 5. **克制配色**：灰色管结构，蓝色管交互，黑色管内容

---

## 执行步骤

### 步骤1：三级标题系统检查（5秒）

**检查标准**：

```
□ 是否只使用3级标题？（H1/H2/H3）
   - 三级系统：清晰简洁
   - 四级以上：层级混乱，小白难以理解结构

□ H1 是否用于主要章节？
   - 正文中的大章节用 H1
   - 所有章节都用 H2 → 层级不清晰
```

**标题层级规范**：

| 标题 | 字号比例 | 字重 | 上边距 | 下边距 |
|------|----------|------|--------|--------|
| H1 | 1.75x正文 | 700粗体 | 48px | 24px |
| H2 | 1.3x正文 | 600中粗 | 32px | 16px |
| H3 | 1.1x正文 | 500中等 | 24px | 12px |

---

### 步骤2：间距检查（10秒）

**检查标准**：

```
□ 正文行高是否为 1.8？
   - 1.8：舒适，适合长时间阅读
   - 1.5：略显拥挤
   - 1.2：过于紧密，难以阅读

□ 普通段落间距是否为 12px？
   - 12px：书籍标准，紧凑舒适
   - 20px+：过于松散，打断阅读节奏

□ 换行/大间距使用 --- 标记？
   - 三个连字符：---
   - 公众号场景：md 转 html 的特殊标识
   - 产生大间距：24px（约1行空白）
   - 用途：段落间需要明显分隔时使用
```

**间距检查表**：

| 场景 | 间距 | 说明 |
|------|------|------|
| 标题后第一段 | 0 | 利用标题下边距 |
| 普通段落间 | 12px | 书籍标准 |
| 列表后段落 | 20px | 12px + 8px（呼吸感） |
| 引用后段落 | 20px | 12px + 8px |
| 代码块后段落 | 20px | 12px + 8px |
| `---` 后段落 | 24px | 换行标记（公众号特殊标识） |

---

### 步骤3：公众号适配检查（5秒）

**为什么这是必做的**：

- 公众号不支持有序列表数字后有空格
- 错误格式会导致排版混乱

**检查标准**：

```
□ 有序列表数字后无空格？
   - 无空格：`1.项目`（公众号显示正常）
   - 有空格：`1. 项目`（公众号不支持）
```

**案例对比**：

```
❌ 错误（有空格）：
1. 第一项
2. 第二项
3. 第三项
→ 公众号显示混乱

✅ 正确（无空格）：
1.第一项
2.第二项
3.第三项
→ 公众号显示正常
```

---

### 步骤4：配色检查（5秒）

**检查标准**：

```
□ 主要是黑白灰？（克制）
   - 近黑色主文本
   - 灰色结构元素
   - 纯白/极浅灰背景

□ 只有链接是蓝色？（交互）
   - 蓝色链接
   - 不要红色/绿色等其他颜色链接

□ 没有多余的彩色？（结构清晰）
   - 不要使用多种彩色
   - 克制使用颜色，只在必要时用蓝色
```

**关键认知**：灰色管结构，蓝色管交互，黑色管内容

---

### 步骤5：列表和引用规范（5秒）

**检查标准**：

```
□ 列表项是否简洁？
   - 每项1-2行
   - 每项5行以上 → 改为段落形式

□ 嵌套列表是否不超过2层？
   - 一级列表 + 二级列表
   - 三层以上嵌套 → 认知负担过重

□ 引用块是否有视觉区分？
   - 左侧边框（3px）
   - 浅灰背景
   - 适当内边距（16-24px）
```

## 视觉自检法（5秒）

**眯起眼睛看**（模糊视觉）：

```
□ 标题层级清晰吗？
   - 能看到3个明显的字号层次

□ 段落之间有明显的"块"感吗？
   - 段落间有视觉分隔

□ 整体看起来紧凑但不拥挤吗？
   - 像纸质书一样舒适

□ 段落分隔处有明显的留白吗？
   - --- 标记产生大间距
```

## 成功标准

### 优秀排版（≥90分）：

```
- 只使用3级标题
- H1标记主要章节
- 行高1.8
- 段落间距一致（12px）
- 换行用 --- 标记（公众号特殊标识）
- 有序列表数字后无空格
- 配色克制（灰色系为主）
- 列表简洁，嵌套≤2层
```

**效果**：接近专业书籍质量，小白阅读舒适

## 常见错误

### 错误1：标题层级过深

```
❌ 使用 H4、H5、H6（层级过深）
   → 改为：用列表、粗体、或重新组织结构

⚠️ H1 之后直接跟 H3（跳级，不规范）
   → 建议：按顺序 H1 → H2 → H3
```

### 错误2：间距不一致

```
❌ 段落间距忽大忽小
   → 改为：统一使用12px

❌ 行高过小（1.2）
   → 改为：统一使用1.8

❌ 没有换行标记
   → 改为：段落分隔用 ---（公众号特殊标识）
```

### 错误3：公众号格式错误

```
❌ 有序列表数字后有空格（1. 项目）
   → 改为：无空格（1.项目）
```

### 错误4：配色过度

```
❌ 使用多种彩色（红橙黄绿青蓝紫）
   → 改为：克制使用颜色

❌ 链接使用红色或其他颜色
   → 改为：链接统一用蓝色
```

## 排版质量评分

```
总分 = (通过项 / 总检查项) × 100

≥90分：优秀排版，接近专业书籍质量
70-89分：良好排版，有改进空间
<70分：排版需要优化，影响阅读体验
```

**必须满足的核心项**（缺一不可）：

```
□ 只使用3级标题
□ 段落间距一致（12px）
□ 行高舒适（1.8）
□ 换行用 --- 标记（公众号特殊标识）
□ 配色克制（灰色系为主）
□ 有序列表数字后无空格
```

---

## 设计理念

```
书籍风格排版 = 紧凑但不拥挤，克制但不单调

三大原则：
1.简洁的层级（3级标题）
2.舒适的间距（行高1.8，段落12px）
3.克制的配色（灰+蓝+黑）

目标：
让小白看得舒服、找得快、不疲劳
```
