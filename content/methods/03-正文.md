# 03-正文：写内容

> **核心原则**
>
> 1. **视觉优先**：视频转发率3倍于文字，抽象概念必有图
> 2. **块化阅读**：段落≤5-7行，1块=1个核心观点
> 3. **术语<10%**：第一段无术语或有解释，小白战略核心要求
> 4. **开头降维**：3秒抓眼球（痛点/对比/反常识）
> 5. **服从小白战略**：当冲突时，小白战略优先

---

## 认知科学四大原理

### 原理1：进化优先级原理（处理速度差异）

```
神经处理是并行的，但速度有差异：

杏仁核(情绪)   20ms     最快  ←─ 生存威胁检测优先
    ║
海马体(记忆)   100ms    次快  ←─ "我遇到过吗?"
    ║
前额叶(理性)   200-500ms 最慢 ←─ 抽象思考成本高
    ║
纹状体(习惯)   重复后固化      ←─ 自动化后最省能量
```

**内容创作应用**：
- ✅ 阶段1必须触发情绪（利用20ms快速通道）
- ✅ 阶段2建立记忆关联（利用100ms通道）
- ✅ 阶段3才进入理性分析（200ms通道）
- ❌ 不能直接讲理性分析（用户情绪未安抚，会启动防御）

### 原理2：网状路径原理（共识侵入）

```
真实情况（网状）：
概念A ←→ 概念B
  ↕      ↕
概念C ←→ 概念D
   \   /
  概念E
```

**共识侵入**（利用已有网状路径）：
```
用户脑中已有：
"对接API" ←→ "报错"
     ↕          ↕
  "文档"  ←→ "调试"

你说："AI对接API总报错？问题在文档"
→ 立即激活用户已有的4个概念
→ 沿着已有路径传播
→ 认知负荷低
```

**引入新概念**（需创建新路径）= 认知负荷高

**内容创作应用**：
1. **不引入新术语**
   - ❌ "MCP是Model Context Protocol"
   - ✅ "MCP像Git，但管理的是AI上下文"

2. **用用户已有的概念组合**
   - ❌ "范式转移"
   - ✅ "这个方法和你之前用的XX不同，区别是..."

3. **建立判断标准，而非新概念**
   - ❌ "这是Agent First设计"
   - ✅ "好文档有3个特征：有Markdown、AI能读懂、错误提示清晰"

### 原理3：认知负荷最小化原理

**底层事实**：
- 工作记忆容量：7±2 chunks
- 大脑占体重2%，消耗20%能量
- 超载 = 放弃（大脑自我保护机制）

**策略A：信息密集化**
```
宁可少而深，不要多而浅：
✅ 10章，每章54行（密集）
❌ 31章，每章14行（分散）
```

**策略B：共识侵入**
```
✅ 激活已有神经路径（省能量）
❌ 创建新概念（耗能量高，被拒绝）
```

### 原理4：信任=期望模型原理

```
Trust(人) = E(结果|这个人的承诺)

三个维度：
1️⃣ 期望值（均值）：信任高 = 期望值高
2️⃣ 方差（确定性）：持续60分 > 忽高忽低（50-80分）
3️⃣ 尾部风险：一次重大失败 → 信任清零

累积慢、清零快：
- 正向更新：Trust × 1.05（慢）
- 负向更新：Trust × 0.2（快）
```

## 执行步骤

### 步骤1：检查表达形式（5分钟）

**传播力排序**：
```
视频(1.0) > 图文(0.7) > 纯文字(0.4)
```

**为什么"实操演示"重要**：

```
纯文字流程：
文字 → 语言理解 → 心理模拟 → 想象画面 → 理解
      ↑_______高认知负荷_______↑

文字+实操流程：
文字 → 语言理解 ↘
                 ↘→ 对照整合 → 理解
实操 → 视觉处理 ↗     ↑
      ↑________低认知负荷________↑
```

**三个关键优势**：
1. **双通道并行处理**：文字（左脑）+ 实操（右脑）
2. **消除"想象成本"**：直接看到操作位置
3. **提供"进度锚点"**："我的画面和视频一样" = 即时反馈

**检查标准**：
```
□ 内容形式选择：
   - 视频 → 转发3倍于文字（最佳）
   - 图文 → 次之
   - 纯文字 → 最差

□ 抽象概念有图解？（必须）
□ 操作步骤有截图？（必须）
□ 5秒测试：看图5秒能说出主要内容？
```

---

### 步骤2：检查块化阅读（10分钟）

**为什么这是最重要的**：
- 信息密度过高 → 大脑自动忽略部分内容
- 即使用户"读完"了，吸收率可能<50%

**检查标准**：

```
□ 段落长度：每段≤5-7行
□ 信息密度：
   - 1块 = 1个核心观点 + 1-2个支撑点
   - 1块 = 2个并列观点（必须紧密相关）
   - 1块 = 3+个并列观点 → 会被忽略
□ 块间分隔：空行、分隔线、小标题
□ 视觉节奏：长块短块交替
```

**自检方法**：
1. 截图你的内容
2. 眯起眼睛看（模糊视觉）
3. 能看到明显的"块"边界吗？

---

### 步骤3：检查术语密度（5分钟）

**小白战略核心要求**：

```
□ 第一段无术语或有解释？（必须）
□ 全文术语占比<10%？（必须）
□ 抽象概念有类比？
□ 用场景化表达？（"当你想XX时" > "功能是XX"）
```

**术语降维表**：

| 技术术语 | 降维表达 | 场景化表达 |
|---------|---------|-----------|
| MCP Server | 让AI连接你的工具 | 当你想让Claude调用滴滴打车时 |
| Prompt Engineering | 如何跟AI说话才有效 | 为什么同样的问题，你问的效果不好？ |
| LLM Agent | 智能助手 | 一个能帮你自动完成工作的AI |
| Token | AI的"阅读成本" | 每个字都要花钱 |

**降维四原则**：
1. 用类比（"像XX一样"）
2. 用场景（"当你想XX时"）
3. 避免术语堆砌
4. 第一段必须无门槛

---

### 步骤4：检查开头3秒（3分钟）

**开头模式**：
```
□ 开头3秒抓眼球？
   - 痛点："你是不是遇到XX问题"
   - 对比："新方法比旧方法快10倍"
   - 反常识："大家都以为X，其实是Y"
   - 平铺直叙："本文介绍XX" → 不行

□ 预告模式：
   - 成果展示："我用X完成Y，节省Z时间" → 好
   - 问答式："你知道X吗？答案是Y" → 不好（浪费时间）
```

**三段式结构**（方法B）：

**1. 开头降维（抓注意力）**
- 目标：3秒内抓住眼球
- 方法：用痛点/对比/反常识制造"认知缺口"
- ❌ 错误："我有个新想法"（教育型）
- ✅ 正确："学N8N花3小时，学Skills只花30分钟，差距在哪？"（共鸣型）

**2. 中间拔高（提供价值）**
- 目标：保留深度和洞察
- 方法：逻辑清晰、案例支撑、层层递进
- 关键：不迁就传播而降低内容质量

**3. 结尾立场（促进转发）**
- 目标：给出明确观点，方便转发
- 方法：提炼一个可表达立场的金句
- ❌ 错误："以上是我的思考"（无立场）
- ✅ 正确："AI时代，不是学工具的语言，而是教工具说人话"（明确立场）

---

### 步骤5：检查结构清晰度（5分钟）

```
□ 结论先行？（金字塔原理）
□ 有小标题？（每3-5段）
□ 逻辑链清晰？（问题→分析→解决方案）
□ 结尾有明确立场？（金句）
```

## 可视化图表设计（强烈推荐）

**原理**：图表 = 认知假肢，不只帮助理解，更是记忆提示器

### 6大认知机制

1. **工作记忆容量限制**：信息压缩到3-5个块
2. **双重编码理论**：图像+文字，同时激活两个系统
3. **认知负荷理论**：降低外在负荷，优化相关负荷
4. **格式塔原理**：接近性、相似性、连续性、闭合性
5. **元认知监控**：理解层→记忆层→应用层
6. **因果推理**：流程可视化，符合A→B→C的思维

### 必须做的（认知基础）

```
□ 符合工作记忆容量：3-5个主要信息块
□ 双重编码：图像 + 文字
□ 降低外在负荷：颜色编码、空间布局代替文字描述
□ 提供对比：新 vs 旧、A vs B（左右对比）
□ 可视化流程：输入→处理→输出
□ 核心价值置顶：3秒内回答"这个东西有什么用"
```

### 强烈推荐（认知增强）

```
□ 使用emoji情绪标记：🎯性能 💰成本 ⚡速度
□ 使用具体数字：95% > "大部分"
□ 使用颜色语义：灰色=旧，亮色=新
□ 左右对比：左侧=传统，右侧=新方法
□ 箭头引导：视线流动 = 逻辑流动
```

### 避免（认知陷阱）

```
□ 装饰性元素（增加认知负荷）
□ 过多颜色（>3-4种）
□ 过小字体（<12pt）
□ 过密信息（信息块间距不足）
□ 复杂图形（需要解释的图标）
```

### 设计三问

每次设计图表前，问自己：
1. **理解层**：用户能不能3秒内看懂核心逻辑？
2. **记忆层**：用户1天后还能不能回忆起关键点？
3. **应用层**：用户能不能用这张图解决实际问题？

## 表格优化

**核心原理**：表格不是唯一选择，选择最适合信息结构的表达形态

**什么时候不用表格**：

| 信号 | 改成 |
|-----|------|
| 两列信息重复（说同一件事） | 列表 |
| 只有 2-3 行 | 列表 |
| 超过 5 行 | 拆分或流程图 |
| 有先后顺序 | 步骤 |
| 问题→方案对照 | ASCII 对比式 |

**ASCII 对比式示例**：

```
❌ Token 放不下      →  ✅ Skills 放得下
❌ 不会自动调用      →  ✅ Skills 自动调
```

---

## 简洁表达格式：「X → Y」

**格式**：`X → Y`

- X = 做什么 / 是什么
- → = 导致 / 意味着 / 结果是
- Y = 效果 / 解释 / 具体内容

**适合场景**：

- 列清单 → 选项多、需要快速扫
- 讲步骤 → 动作+结果一行搞定
- 做对比 → 左右对照，差异明显

**不适合场景**：

- 讲故事 → 需要情感铺垫
- 深度分析 → 需要展开论证

## 卖点 vs 说明书

**原则**：用"卖点"语言，不用"说明书"语言

| | 卖点（用这个） | 说明书（避免） |
|--|--------------|--------------|
| 视角 | 用户收益 | 功能描述 |
| 语言 | 口语、有画面 | 技术、抽象 |
| 情绪 | 期待感 | 中性 |

**案例对比**：

```
说明书风格：
- 放得下
- 自动调
- 全局同步

卖点风格：
- 放得下 → 整个图书馆都能塞进去
- 自动调 → 你说"检查"，它自己知道用什么
- 能迭代 → 写一次，到处用，越用越强
```

## 四种传播方法

### 方法A：共识传播极致

**策略**：不创造新观点，而是把"模糊共识"说清楚

**执行步骤**：
1. 找共识：大家都隐约认同，但没人说清楚的
2. 提炼：用一句话/一个公式说透
3. 验证：用案例/数据证明
4. 传播：包装成易记的金句/框架

### 方法B：开头降维-中间拔高-结尾立场

（见步骤4详细说明）

### 方法C：共识锚点法

**策略**：用"已有共识"作为跳板，引导到"新观点"

**四步结构**：
1. 先说大家都认同的（建立信任）
2. 提出一个反常识的观察（制造好奇）
3. 用数据/案例证明（降低怀疑）
4. 给出新的解释框架（重构认知）

### 方法D：反向切片法

**策略**：将多个热点话题作为数据样本，反向提炼成系统化内容

**核心逻辑**：
```
传统切片：完整内容 → 切成片段 → 分发传播
反向切片：网络热点（碎片） → 整合提炼 → 深度输出
```

**双重价值**：
- 传播优势：热点自带流量
- 需求洞察：成熟内容 = 市场验证过的真实需求
