# 02-大纲：写结构

> **核心原则**
>
> 1. **4阶段完整**：认知→兴趣→决策→行动，缺一不可
> 2. **认知阶段3理由**：最常见错误是跳过或动机薄弱（一票否决）
> 3. **承诺设计**：好承诺 = 具体 + 可信 + 有吸引力
> 4. **覆盖≥3类人群**：小白战略核心要求
> 5. **最佳密度40-60行/章**：太密过载，太稀分散
> 6. **基础80%+进阶20%**：服从小白战略

---

## 执行步骤

### 步骤1：检查认知阶段（10分钟）

**为什么这是最重要的**：

- 小白不是"不会用"，而是"不知道为什么要用"
- **跳过认知阶段 = 违反小白战略 = 一票否决**

**检查标准**：

```
□ 有"为什么"章节？（必须有）
□ 列出了3个以上理由？（必须）
□ 用户看完觉得"我需要这个"？（测试标准）
```

**3理由模板**：

```
为什么要用X？

1.痛点1：你是不是遇到[具体问题]？
  - 现状：大部分人用[旧方法]，但[缺点]
  - 改变：用X可以[好处]

2.痛点2：你是不是想[具体目标]？
  - 现状：用[旧方法]需要[学习成本]
  - 改变：X让你[零门槛]就能做到

3.痛点3：你是不是担心[具体顾虑]？
  - 现状：[旧方法]存在[风险]
  - 改变：X提供[保障]
```

---

### 步骤2：检查兴趣阶段（5分钟）

**检查标准**：

```
□ 列出了3-5个使用场景？（必须）
□ 用了"你是不是..."句式？（推荐）
□ 覆盖≥3类人群？（小白战略要求）
```

**场景模板**：

```
谁应该学X？

场景1：[人群1]
- 你是不是经常[具体场景]？
- X能帮你[具体好处]

场景2：[人群2]
- 你是不是想[具体目标]？
- X能让你[具体结果]

场景3：[人群3]
- 你是不是遇到[具体问题]？
- X能解决[具体方案]
```

### 步骤3：承诺设计（10分钟）

**承诺公式**：

```
好承诺 = 具体 + 可信 + 有吸引力

其中：
- 具体 = 可视化的结果（不是"变好"，而是"拥有X"）
- 可信 = 可验证的承诺（读者相信能做到）
- 有吸引力 = 描绘理想状态（触发渴望情绪）
```

**承诺的3个底层作用**：

```
作用1：触发情绪（基于认知科学原理1）
承诺 = 描绘一个"理想状态"
  ↓
激活大脑的奖励系统（多巴胺）
  ↓
产生"我想要这个"的冲动
  ↓
继续阅读/购买

作用2：降低认知负荷（基于认知科学原理3）
没有承诺：读者要自己判断"值不值得看"
  ↓
需要调用前额叶（200ms）
  ↓
认知负荷高 → 可能放弃

有承诺：直接告诉你"看完能得到X"
  ↓
不需要判断，直接决策
  ↓
认知负荷低 → 继续看

作用3：建立期望（基于认知科学原理4）
信任 = 期望契合度 = min(实际体验, 承诺)

如果：承诺 > 实际体验 → 失望 → 取关
如果：承诺 ≈ 实际体验 → 满意 → 信任
如果：承诺 < 实际体验 → 惊喜 → 转发
```

**承诺的4个层级**：

| 层级 | 承诺类型 | 例子 | 适用场景 |
|------|----------|------|----------|
| **弱承诺** | 信息型 | "今天讲讲数据分析" | 进阶内容（老粉丝） |
| **中承诺** | 技能型 | "学会5个数据分析方法" | 基础教程（60%） |
| **强承诺** | 结果型 | "等于拥有一个数据分析团队" | 爆款内容 |
| **超承诺** | 改变命运型 | "改变你的人生" | 慎用（难以兑现） |

**承诺甜点区**：

```
承诺 = 实际价值 × 0.8

例子：
- 实际能提供的价值：学会5个数据分析方法
- 承诺："学会我的方法，你等于拥有了一个数据分析团队"

判断：
- 如果5个方法真的能解决团队80%的工作 → 承诺合理 ✅
- 如果5个方法只是入门 → 承诺过大 ❌
```

**4个承诺模板**：

```
模板1：时间型承诺（适合教程）
[时间] + [动词] + [结果]
- "3分钟看懂MCP"
- "10分钟学会Claude Code"
- "1小时掌握数据分析"

模板2：结果型承诺（适合产品）
[学会/掌握] + [方法] + [等于] + [理想状态]
- "学会我的方法，等于拥有一个数据分析团队"
- "掌握这个框架，等于有了产品经理的思维"

模板3：对比型承诺（适合解决痛点）
[不需要X] + [也能做Y]
- "不需要团队，也能做专业数据分析"
- "不需要学代码，也能开发MCP"

模板4：数字型承诺（适合系统内容）
[数字] + [具体方法] + [解决问题]
- "5个方法，搞定数据分析"
- "3步流程，快速上手MCP"
```

**承诺检查**：

```
□ 承诺具体吗？（可视化的结果）
□ 承诺可信吗？（能验证）
□ 承诺有吸引力吗？（触发渴望）
□ 承诺能兑现吗？（承诺 ≈ 实际价值 × 0.8）
```

### 步骤4：检查决策阶段（10分钟）

**检查标准**：

```
□ 安装步骤详细？（截图+提示词）
□ 强调"简单"？（"3分钟"、"复制粘贴"）
□ 有明确承诺？
```

---

### 步骤5：检查行动阶段（5分钟）

**检查标准**：

```
□ 有进阶内容？（给深度用户价值）
□ 路径清晰？（基础→进阶→高级）
□ 标注层级？（"必学" vs "可选"）
```

**进阶内容模板**：

```
进阶玩法（可选）

必学（80%用户需要）
- [ ] 基础功能A
- [ ] 基础功能B

推荐（20%用户需要）
- [ ] 进阶功能D（适合[场景]）

高级（5%用户需要）
- [ ] 高级功能F（仅适合[专业场景]）
```

---

### 步骤6：检查内容密度（5分钟）

**检查标准**：

```
□ 章节数≤15章？（避免信息过载）
□ 单章40-60行？（最佳密度）
□ 标注了重点？（"必读"、"必学"）
```

**密度标准**：

```
40-60行/章：最佳密度（CLI教程54行/章，1000+赞）
20-40行/章：稍稀疏，需要微调
<20行/章：过于分散（Codex教程14行/章，数据一般）
>80行/章：过于密集，认知过载
```

## 承诺位置设计

**位置1：标题**
```
作用：决定点击率
承诺强度：中-强
例子："3分钟看懂MCP"（中承诺）
```

**位置2：开头第一段**
```
作用：决定继续阅读率
承诺强度：强
例子：
"看完这篇文章，你将学会5个数据分析方法，
以后不需要团队，也能做专业级的分析。"
```

**位置3：中间转折点**
```
作用：防止读者流失
承诺强度：中
例子：
"掌握了前面3个方法，接下来我要告诉你
最关键的2个技巧，这是团队都在用的。"
```

**位置4：结尾CTA**
```
作用：转化行动
承诺强度：强
例子：
"现在加入我的课程，你将获得：
- 完整的数据分析系统
- 1对1的答疑支持
- 等于拥有一个专业团队"
```

## 成功案例

### CLI教程（1000+赞）

- 认知阶段：3个充分理由
- 兴趣阶段：3类人群
- 决策阶段：详细步骤+承诺"10分钟学会"
- 行动阶段：进阶玩法
- 内容密度：10章、54行/章
- 承诺设计：中承诺"10分钟学会"（能兑现）

**结果**：26h投入 → 1000+赞 + 250粉丝

### FastMCP系列（日均+91粉）

- 承诺设计："3分钟看懂MCP"
- 具体：3分钟（时间明确）
- 可信：入门级，3分钟够了
- 吸引力：看懂 = 理解新技术

**结果**：日均+91粉（系统化内容）

## 失败案例

### 插件教程

- 跳过"为什么"章节
- 目标客户单一（<3类人群）

**教训**：必须有认知阶段，且有3个以上理由

### Codex教程

- 只说"10块钱"（1个弱理由）
- 31章、14行/章（过于分散）
- 无明确承诺

**教训**：
- 认知阶段要3个以上充分理由
- 内容密度40-60行/章最佳
- 需要有明确承诺

### 超承诺的风险

- 假设承诺："这个方法改变我的人生"
- 具体：❌（改变人生太模糊）
- 可信：❌（很难验证）
- 吸引力：✅（但过度）

**风险**：
- 期望过高 → 很难满足 → 失望
- "标题党"嫌疑 → 失去信任
