# 01-选题：写之前

> **核心原则**
>
> 1. **三维匹配**：价值 = 人×情境×信息，任何一维为0，价值为0
> 2. **先验调研**：不做调研=闭门造车（必做）
> 3. **市场洼地**：搜索<10条=金矿，>100条=晚期（换主题）
> 4. **信息增量≥70%**：重复内容=浪费时间=取关
> 5. **24-48h完成**：抢占先机，成为小白第一个看到的教程
> 6. **覆盖≥3类人群**：小白战略核心要求

---

## 第一性原理：三维匹配法

### 内容价值的本质

```
价值 = 人 (Who) × 情境 (Context) × 信息 (What)

任何一维为0，价值就为0：
- 人不对 × 情境再痛 × 信息再好 = 0
- 人对了 × 情境不痛 × 信息再好 = 0
- 人对了 × 情境很痛 × 信息没用 = 0
```

### 三个维度定义

**维度1：人 (Who)**
```
人 = 认知结构 + 技能基础 + 资源约束

- 认知结构：知道什么，不知道什么
- 技能基础：会做什么，不会做什么
- 资源约束：有什么，缺什么（时间/金钱/注意力）
```

**维度2：情境 (Context)**
```
情境 = 触发事件 + 当前状态 + 期望状态 + 行动阻碍

- 触发事件：什么事让他需要这个信息
- 当前状态：他现在在哪里
- 期望状态：他想到哪里
- 行动阻碍：什么阻止他到达
```

**维度3：信息 (What)**
```
信息 = 信息增量 + 认知门槛 + 行动成本

- 信息增量：缩短"当前→期望"的距离
- 认知门槛：理解需要多少背景知识
- 行动成本：应用需要多少资源
```

### 三元组定义模板

```markdown
## 内容三元组定义

### 人 (Who)
**认知结构**：
- 知道：[他知道什么概念/工具/方法]
- 不知道：[他不知道什么，需要解释的]

**技能基础**：
- 会做：[他会什么技能]
- 不会做：[他不会什么，需要降低门槛的]

**资源约束**：
- 时间：[每天/每周有多少时间]
- 金钱：[愿意为此付费吗？多少？]
- 注意力：[能集中多久？]

---

### 情境 (Context)
**触发事件**：
- 什么事：[具体的事件]
- 什么时候：[时间节点]
- 情绪状态：[焦虑/兴奋/困惑...]

**当前状态**：[他现在在哪里]

**期望状态**：[他想到哪里]

**行动阻碍**：
- 什么阻止他：[具体障碍]
- 为什么阻止：[根本原因]

---

### 信息 (What)
**信息增量**：
- 如何缩短距离：[提供什么新信息/新方法]
- 为什么有效：[逻辑]

**认知门槛**：
- 需要什么背景：[前置知识]
- 如何降低门槛：[用类比/案例/降维表达]

**行动成本**：
- 时间成本：[看完需要多久？应用需要多久？]
- 金钱成本：[免费/付费？]
- 精力成本：[需要多专注？]
```

## 执行步骤

### 步骤1：三维匹配检查（5分钟）

**检查清单**：

```
人 × 信息：
□ 信息的认知门槛 ≤ 人的认知结构（他能看懂吗？）
□ 信息的技能要求 ≤ 人的技能基础（他能做到吗？）
□ 信息的行动成本 ≤ 人的资源约束（他愿意付出吗？）

情境 × 信息：
□ 信息增量能缩短"当前→期望"的距离（真的有用吗？）
□ 信息能解除行动阻碍（解决核心障碍吗？）
□ 信息的时效性匹配触发事件的紧迫性（他现在就需要吗？）

人 × 情境：
□ 这个情境对这类人是真实存在的（不是臆想的吗？）
□ 期望状态对这类人是可达的（目标太高他做不到）
□ 触发事件的频率足够高（只有1%的人遇到=受众太窄）
```

**评分标准**：
- **9个全满足** ✅ 完美匹配，开始创作
- **7-8个满足** ⚠️ 可以做，但需要优化某个维度
- **≤6个** ❌ 匹配度低，重新设计三元组

---

### 步骤2：先验调研（10分钟）

**检查标准**：

```
□ 去平台看过热点趋势？（必做）
   - 小红书 □
   - 公众号 □
   - 知乎 □

□ 记录了3个热门话题？
□ 查看了用户评论（痛点在哪里）？
```

---

### 步骤3：市场洼地识别（5分钟）

**快速判断标准**：

**搜索结果数量**：

```
- <10条 → 金矿（全力投入）
- 10-50条 → 银矿（值得做，但要快）
- >100条 → 晚期（建议换主题）
```

**用户反馈**：

```
- 有人说"很少人讲" → 金矿信号
- 有人说"不知道" → 需求信号
- 主动询问的人多 → 验证需求
```

**完成时间**：

```
- 24-48h内 → 快速抢占
- 1周内 → 中等
- >1周 → 可能错过窗口
```

---

### 步骤4：信息增量检查（5分钟）

**检查标准**：

```
□ 新信息≥70%？（必须）
   - 第1篇：相对市场
   - 第2篇+：相对自己第1篇

□ 列出了市场已有内容？
□ 找出了自己的独特价值？
□ 是真实洞察还是搬运？
```

**如果是系列第2篇+**：

```
需要删除的重复内容：
- [段落/章节名称]

第2篇聚焦的独特价值：
- [独特价值1]
- [独特价值2]
```

---

### 步骤5：圈层交集检查（5分钟）

**检查标准**：

```
□ 会关心的圈层≥3个？（小白战略要求）
□ 外圈人能看懂？
□ 外圈人愿意转发？
```

## 需求洞察的四种方法

### 方法概览

| 方法 | 本质 | 优势 | 局限 | 适用场景 |
|------|------|------|------|---------|
| **成熟内容分析** | 被动观察 | 市场验证过，可靠性高 | 滞后，已有竞争 | 进入新领域，学习规律 |
| **用户调研** | 主动询问 | 直接接触用户 | 用户说的≠想要的 | 优化现有产品 |
| **演讲/直播** | 实时反馈 | 即时互动 | 样本偏差 | 测试新想法 |
| **内容反馈收集** | 发布后观察 | 真实行为，大样本 | 需要已有内容 | 持续迭代 |

### 方法A：成熟内容需求分析

**执行步骤**：

**第1步：收集成熟内容（3-5个）**
- 同一领域/同一类问题
- 都已经被市场验证（高流量/高关注）
- 跨时间验证（不是同一时期的偶然热点）

**第2步：表面分析**
- 这些内容在讲什么？（主题）
- 解决了什么问题？（表面需求）
- 目标用户是谁？（受众）

**第3步：共性提炼**
```
共同点分析：
1. 功能共性：都在做什么
2. 需求共性：都在解决什么痛点
3. 价值共性：为用户创造了什么价值
```

**第4步：需求公式化**
```
需求公式 = 核心价值 × 放大因素

核心价值：用户为什么需要这个
放大因素：什么让这个需求更强烈
```

**第5步：场景迁移**
- 选题方向：还有哪些领域有相同需求？
- 趋势预判：下一个满足这个需求的工具是什么？
- 内容创新：如何用新角度讲这个需求？

### 方法B：用户调研

**关键原则**：

```
原则1：观察行为 > 听取意见
❌ "你觉得这个功能好吗？" → 客套话
✅ "你上次遇到XX问题时，怎么解决的？" → 真实需求

原则2：问"为什么" 5次
用户："我想学ChatGPT"
→ 为什么？→ 写文章太慢
→ 为什么？→ 不知道怎么组织思路
→ 为什么？→ 没学过写作
→ 底层需求：降低专业写作门槛

原则3：找痛点 > 找需求
❌ "你想要什么功能？"
✅ "你最近遇到的最大困难是什么？"
```

### 方法C：演讲/直播反馈

**关键观察点**：
- 哪里有人提问 → 痛点/困惑点
- 哪里有人点头/刷屏 → 共鸣点
- 哪里有人反驳 → 非共识/创新点
- 用户最常问的问题 → 最大需求

### 方法D：内容反馈收集

**关键信号**：
- 高频问题 → 需求缺口
- 困惑点 → 理解门槛
- 延伸需求 → 系列机会
- 反对声音 → 边界条件
- 意外共鸣 → 爆款种子

## 爆款概率公式

```
爆款概率 = 信息源质量 × 传播链路效率 × 市场时机

展开：
爆款概率 = (洞察质量 × 表达能力)
         × (触达率 × 理解率 × 价值感知率 × 传播率)
         × 市场时机

其中：
洞察质量 = 用户反馈质量 × 洼地识别速度 × 时间节点
市场时机 = 1 / (1 + 市场内容数量)
```

**核心洞察**：
- 爆款不是单一因素，是**信息传递全链路**的乘积
- 任何一个环节=0，结果就=0
- 市场时机是**乘法因子**（早期1.0，晚期0.3）

## 失配诊断表

**人维度失配**

表现：用户看不懂/学不会/做不到

可能原因：
- 认知门槛太高 → 降维表达
- 技能要求太高 → 拆解步骤、提供模板
- 资源成本太高 → 提供快速方案

**情境维度失配**

表现：用户没兴趣/不觉得有用/不行动

可能原因：
- 触发事件不存在 → 重新定义情境
- 距离太小（已在期望状态）→ 提高期望
- 阻碍不痛 → 放大问题严重性

**信息维度失配**

表现：用户看了但没收获/没法用

可能原因：
- 信息增量为0 → 提供新视角/新方法
- 信息不可执行 → 给步骤/给工具/给模板
- 信息不适配 → 匹配他的具体问题

## 成功案例

### FastMCP系列（日均+91粉丝）

- 三维匹配：AI编程者 × MCP学习困境 × 5分钟教程
- 先验调研：发现MCP热度上升
- 市场洼地：搜索<10条（金矿）
- 市场早期：快速介入（24-48h）
- 信息增量：系统化系列，新信息充足
- 圈层交集：技术圈+产品圈+创业圈

**结果**：30h投入 → +635粉丝

## 失败案例

### MCP晚期文章

- 搜索>100条（市场饱和）
- 信息增量低（<30%）
- 未做先验调研
- 市场时机系数0.3（vs早期1.0）

**教训**：市场时机 > 内容质量

**结果**：阅读-67%
